Harim Choi Portfolio Site Plan

Personal Project Portfolio Web Application

Harim Choi

Overview

Purpose

The purpose of this website is to present my personal portfolio, resume information, and selected programming projects in one organized place. This site will help visitors quickly understand my technical skills, project experience, and the type of work I am interested in building.

Audience

The intended audience for this site includes potential employers, internship recruiters, professors, classmates, and project teammates. The site is designed for people who want to see my programming skills, project history, and examples of my work without needing to search through multiple links.

Dynamic Elements

JavaScript will be used to organize projects by category and tags. The project data will be stored in an array of objects, and JavaScript will create project cards dynamically. Users will be able to filter projects by categories such as Web, Software, Data / ML, Backend, Cloud / Linux, and Engineering. Users will also be able to search projects by keyword or skill, sort the project list, and open a detail section for each project.

Pseudocode

  1. Create an array of project objects.
  2. Each project object will include a title, category, date, description, skills, image, and link.
  3. When the page loads, select the project container from the HTML.
  4. Loop through the project array.
  5. Create a project card for each project using JavaScript.
  6. Display the project title, image, category, description, skills, and a details button.
  7. Create category filter buttons such as All, Web, Software, Data / ML, Backend, Cloud / Linux, and Engineering.
  8. When a filter button is clicked, check which category the user selected.
  9. If the selected category is All, display every project.
  10. Otherwise, use the filter method to display only projects that match the selected category.
  11. Create a search input box.
  12. When the user types in the search box, compare the search text with the project title, description, and skills.
  13. Display only the projects that match the search text.
  14. Create a sort dropdown.
  15. When the user selects a sort option, sort the project array by newest date or project name.
  16. Re-render the project cards after filtering, searching, or sorting.
  17. Add a View Details button to each card.
  18. When the View Details button is clicked, display more information about the selected project.

Content

Home Page Content

Hi, I’m Harim Choi. I am a computer science student building projects in web development, software development, databases, APIs, cloud computing, and machine learning. This portfolio shows selected projects I have worked on, the technologies I used, and what I learned from each project.

The home page will include a short introduction, a skills summary, and featured project cards. It will also include a button that leads visitors to the full projects page.

Projects Page Content

The projects page will show a list of my programming projects. Each project card will include the project title, category, short description, technologies used, and a button to view more details.

The project categories will include Web, Software, Data / ML, Backend, Cloud / Linux, and Engineering. These categories will help visitors understand my skills across different technical areas.

Project Category Content

  • Web: HTML, CSS, JavaScript, responsive design, and frontend layouts.
  • Software: C++, C#, Java, data structures, algorithms, and console applications.
  • Data / ML: Python, data analysis, machine learning models, metrics, and charts.
  • Backend: SQL, PostgreSQL, Node.js, Express, APIs, and server-side data.
  • Cloud / Linux: Firebase, Firestore, GitHub Pages, VPS, PM2, and Linux deployment.
  • Engineering: robotics concepts, product ideas, embedded systems, and mechanical design interests.

Example Project Content

  • BYU-I Professor Finder: A campus project that helps users find professor information, office locations, buildings, and office hours.
  • Cloud Reaction Game Leaderboard: A cloud database project using Firebase Firestore to store and display game scores.
  • Bike Rental Machine Learning Project: A machine learning project that predicts bike rental demand using weather and time data.
  • Housing Price Prediction Project: A machine learning project that uses housing data to predict prices and compare model performance.

Contact Section Content

The contact section will include simple ways for visitors to connect with me, such as GitHub, LinkedIn, and email. This section will be simple and professional.
